# Product UI Workflow

Дата: 2026-04-23

## Постоянный алгоритм выполнения задач

Этот алгоритм нужен для Codex и других ИИ-агентов, которые продолжают работу в репозитории.

- [ ] Принять задачу и кратко зафиксировать цель.
- [ ] Создать или обновить markdown-план по задаче.
- [ ] Разбить работу на этапы и подэтапы с чекбоксами.
- [ ] Выполнить анализ текущей реализации перед правками.
- [ ] Выполнить один подэтап.
- [ ] Провести self-review выполненного подэтапа.
- [ ] Запустить релевантные тесты или smoke-проверку подэтапа.
- [ ] Отметить подэтап выполненным в документе.
- [ ] После завершения всех подэтапов этапа провести review этапа.
- [ ] Запустить релевантные тесты этапа.
- [ ] Отметить этап выполненным в документе.
- [ ] Повторять цикл, пока все этапы не выполнены.
- [ ] Перед финалом прогнать общую проверку, build/typecheck по необходимости.
- [ ] Сделать осмысленный commit.
- [ ] Push выполнить только если задача или пользователь это подразумевает.

Правило интерфейса: не писать маркетинговые поясняющие блоки внутри продукта. Подсказки должны быть короткими, прикладными и появляться только там, где пользователь реально принимает решение.

## Задача: Единая структура чата и инструментов

Цель: привести продукт к структуре, где слева всегда видны чаты, сверху доступны инструменты, а выбор моделей сделан через единый popup с вкладками для текстовых и image-моделей.

- [x] Найти текущую реализацию левого sidebar в чате.
- [x] Найти текущую реализацию `/images`.
- [x] Найти текущие endpoints моделей текста и изображений.
- [x] Найти место, где чат запускает image generation.
- [ ] Зафиксировать вывод анализа после первого прохода.

Вывод: чат уже содержит нужный sidebar со списком чатов, но `/images` использует отдельный `AppShell`, где слева нет списка чатов. На странице `/images` выбор image-модели есть, но это обычный select, а не popup.
